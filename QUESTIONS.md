# Пул вопросов по резюме

Этот файл описывает фиксированный пул вопросов, которые backend может отправлять клиенту в сценарии парсинга резюме.

## Общие правила

- Вопросы формируются НЕ из LLM, а из фиксированного пула.
- Вопросы со свободным текстом НЕ отправляются.
- Вопросы отправляются только при отсутствии ключевых полей:
  - `areas`
  - `work_format`
  - `experience_level`
  - `salary_min`
- Максимум за один раунд: `3` вопроса.

## Формат вопроса

Поля объекта `Question`:
- `id` — машинный идентификатор поля
- `text` — текст вопроса
- `type` — тип вопроса
- `options` — список вариантов

## Пул вопросов

### 1) Регион

- `id`: `areas`
- `type`: `single_choice`
- `text`: `Выберите регион поиска работы`
- `options` (фиксированный набор, валидный по HH):
  - `Москва`
  - `Санкт-Петербург`
  - `Казань`
  - `Новосибирск`
  - `Екатеринбург`
  - `Нижний Новгород`
  - `Самара`
  - `Краснодар`
  - `Ростов-на-Дону`
  - `Россия`

Примечание: значение должно быть регионом из HH-справочника.

### 2) Формат работы

- `id`: `work_format`
- `type`: `single_choice`
- `text`: `Выберите предпочитаемый формат работы`
- `options`:
  - `Удаленно`
  - `Гибрид`
  - `Офис`

Маппинг ответа (RU -> внутреннее значение):
- `Удаленно` -> `remote`
- `Гибрид` -> `hybrid`
- `Офис` -> `office`

### 3) Зарплата

- `id`: `salary_min`
- `type`: `numeric_input`
- `text`: `Укажите ожидаемую зарплату в рублях (число без пробелов), либо выберите один из вариантов`
- `options` (пресеты):
  - `50000`
  - `100000`
  - `150000`
  - `200000`

Примечание: для зарплаты допускается ввод числа пользователем (исключение из правила без free text).

### 4) Опыт работы

- `id`: `experience_level`
- `type`: `single_choice`
- `text`: `Укажите ваш опыт работы`
- `options`:
  - `Нет опыта`
  - `1-3 года`
  - `3-6 лет`
  - `6+ лет`

Маппинг ответа (RU -> внутреннее значение HH API):
- `Нет опыта` -> `noExperience`
- `1-3 года` -> `between1And3`
- `3-6 лет` -> `between3And6`
- `6+ лет` -> `moreThan6`

## Валидация ответов

### areas
- Принимается только значение, которое сопоставляется с HH-списком регионов.
- Несвязанный/невалидный ответ игнорируется (вопрос остается в очереди).

### work_format
- Принимаются русские варианты ответа из options и их текстовые синонимы, которые нормализуются к:
  - `remote`
  - `hybrid`
  - `office`
- Несвязанный/невалидный ответ игнорируется.

### salary_min
- Значение должно быть числом в диапазоне `1000..10000000`.
- При невалидном значении backend возвращает ошибку:
  - `invalid answer for salary_min: expected numeric value in range 1000..10000000`

### experience_level
- Принимаются русские варианты ответа из options и текстовые синонимы.
- Значение нормализуется в одно из:
  - `noExperience`
  - `between1And3`
  - `between3And6`
  - `moreThan6`
- Невалидный/несвязанный ответ игнорируется (вопрос остается в очереди).

## Логика для клиента

1. Получить `questions[]` из `UploadAndParseResume` или `AnswerResume`.
2. Показывать только вопросы из этого пула.
3. Отправить ответы в `AnswerResume`.
4. Если ответ невалиден:
   - для `salary_min` — получить явную ошибку и показать пользователю;
   - для `areas`/`work_format` — вопрос придет снова, пока не будет валидного ответа.
5. Повторять цикл, пока `status != completed`.
